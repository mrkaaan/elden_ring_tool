# 艾尔登法环存档管理器 (v2.0) / Elden save

基于ModEngine的开发的老头环的存档管理工具。

## 功能

- **存档管理**：在 `saves` 目录中列出所有存档备份，支持一键导入或导出当前游戏存档。
- **路径检测**：自动检测游戏存档路径 (`%AppData%\Roaming\EldenRing`)，若失败可手动指定。
- **配置记忆**：“直接覆盖”等选项的设置会被保存至 `config.json`，重启后状态不变。
- **启动游戏**：点击“启动游戏”按钮，执行同目录下的 `launchmod_eldenring.bat` 文件来启动游戏。程序会检查该文件是否存在，并据此启用或禁用按钮。

## 变更

由于发现了更加好用的ModEngine，工具的核心定位已发生了变化：

- 移除了与“无缝联机MOD”相关的所有配置和管理功能（如密码、死亡惩罚设置等）。
- 启动游戏的功能从执行游戏目录下的 `ersc_launcher.exe` 改为执行同目录下的 `launchmod_eldenring.bat`。
- 现在使用 **ModEngine** 作为MOD管理器来加载和运行游戏，本工具不再处理任何MOD文件。

## 布局说明

### 界面布局说明
本工具界面分为三个主要区域，整体采用无分组框标题的简洁设计，依靠控件层级区分功能。

**信息区域**
- **存档路径**：不展示真实路径。根据状态显示不同文本与颜色：
    - 灰色：“正在检测...”
    - 绿色：“存档路径 ✓ (有存档)”
    - 橙色：“存档路径 ✓ (无存档)”
    - 红色：“存档路径 ✗ (点击手动定位)”
    - *交互*：鼠标悬停显示完整路径提示（Tooltip），单击打开任务管理器指定路径。
- **工具路径**：显示文本“工具路径”（灰色），不展示真实路径。
    - *交互*：样式同“刷新”，鼠标悬停变蓝且鼠标样式改变，单击打开任务管理器指定路径。
- **刷新**：显示文本“刷新”（灰色）。
    - *交互*：Label模拟链接样式，鼠标悬停变蓝且鼠标样式改变，点击触发刷新。

**存档管理区域**
- **第一行**：静态 Label“选择存档：”（注意：仅为普通 Label，非分组框标题）。
- **第二行**：下拉选择框，用于选择具体存档。
- **第三行**：
    - 左侧 Label：状态提示（如“找到1个存档”及合法性校验），颜色逻辑同信息区域（灰/橙/红/绿）。
    - 右侧 Label：“刷新”，交互样式同信息区域的刷新按钮。
- **第四行**：两个复选框（Checkbutton），右侧附带文字说明：
    - 自动保存
    - 直接覆盖
- **第五行**：两个功能按钮：
    - 左侧：“导入存档”
    - 右侧：“导出存档”

**启动游戏区域**
- **第一行**：核心按钮“启动游戏”。
- **第二行**：
    - 左侧 Label：状态提示（如“未找到启动文件”），颜色逻辑同上。
    - 右侧 Label：“刷新”，交互样式同上。

### 控件排版规范
根据界面层级，统一字号与字重标准如下：

| 控件类型 | 建议字号 | 字重 | 设计意图 |
| :--- | :--- | :--- | :--- |
| **主按钮** (启动游戏) | 11pt | Bold | 视觉重心，引导用户完成最终目标。 |
| **区域标题** (选择存档:) | 10pt | Bold | 弥补无分组框的缺陷，建立区域层级。 |
| **交互标签** (刷新/路径) | 10pt | Bold | 模拟超链接样式，强调“可点击”属性。 |
| **输入控件** (下拉框/按钮) | 10pt | Normal | 提升可读性，增加点击热区的视觉舒适度。 |
| **普通状态文本** | 10pt | Normal | 替代原有的9pt，作为标准阅读字号。 |
| **辅助说明** (复选框文字) | 9pt | Normal | 仅用于非常次要的备注信息。 |
