# 艾尔登法环存档与MOD管理工具

## 开发背景

我是一位艾尔登法环玩家，经常和朋友一起去网吧玩游戏。由于网吧电脑不保存个人数据，每次游玩都需要：
1. 导入个人游戏存档
2. 安装无缝联机MOD
3. 修改MOD配置文件（如联机密码）
4. 使用MOD启动器启动游戏

这些手动操作繁琐且容易出错，因此开发了这个集成化工具，旨在提供一键式的存档和MOD管理功能。

**开发能力说明**：我个人的编程能力有限，这个工具的大部分代码是在AI助手（DeepSeek）的帮助下完成的。通过多次迭代和调试，逐渐完善了各项功能。

## 功能详细说明

### 1. 路径管理
- **自动检测**：从Windows注册表读取Steam安装路径，自动查找艾尔登法环游戏目录
- **手动定位**：当自动检测失败时，可点击路径标签手动选择目录
- **状态指示**：
  - 绿色✓：路径存在且正常
  - 红色✗：路径不存在，可点击手动定位
  - 存档路径额外显示存档状态（有存档/无存档）

### 2. MOD管理（无缝联机MOD专版）
- **MOD源文件状态**：检查`mods/seamless_coop`目录下的MOD文件完整性
- **已安装MOD状态**：检查游戏目录中是否已正确安装MOD
- **一键导入**：将MOD文件复制到游戏目录，支持覆盖选项
- **配置管理**：
  - 联机密码：4位数字，修改后立即生效到`ersc_settings.ini`
  - 死亡惩罚：通过单选按钮设置（启用/禁用）
- **实时读取**：直接从INI文件读取配置，确保显示与实际文件一致

### 3. 存档管理
- **存档浏览器**：下拉列表显示`saves`目录中的所有存档文件夹
- **合法性验证**：
  - 检查存档文件夹结构
  - 验证SteamID文件夹（17位数字）和ER存档文件
  - 支持两种结构：带`EldenRing`文件夹和不带
- **导入存档**：
  - 支持直接覆盖选项（默认选中）
  - 导入前自动备份当前存档（仅当存档目录非空时）
  - 检测游戏是否运行，避免文件占用问题
- **导出存档**：将当前游戏存档导出为带时间戳的备份
- **智能备份**：
  - 自动备份：导入时创建，后缀`_auto_backup`
  - 手动备份：导出时创建，后缀`_manual_backup`
  - 备份内容：整个`EldenRing`文件夹结构

### 4. 状态指示系统
- **路径状态**：显示路径存在性和详细信息
- **MOD状态**：显示MOD文件完整性和安装状态
- **存档状态**：显示选中存档的合法性状态
- **工具状态栏**：实时显示操作状态和结果

## 存档目录结构和备份策略

### 目录结构
```
工具根目录/
├── saves/                    # 存档备份目录（自动创建）
│   ├── YYYYMMDD_HHMMSS_auto_backup/    # 自动备份（导入时创建）
│   │   └── EldenRing/        # 完整的存档文件夹
│   │       ├── 76561198925622664/      # SteamID文件夹（17位数字）
│   │       │   ├── ER0000.sl2          # 游戏存档文件
│   │       │   ├── ER0000.co2          # MOD存档文件
│   │       │   └── ER0000.co2.bak      # 备份文件
│   │       └── GraphicsConfig.xml      # 图形配置（可选）
│   └── YYYYMMDD_HHMMSS_manual_backup/  # 手动备份（导出时创建）
│       └── EldenRing/        # 结构同上
├── mods/                     # MOD存储目录（自动创建）
│   └── seamless_coop/        # 无缝联机MOD文件
│       ├── SeamlessCoop/     # MOD核心文件夹
│       │   └── ersc_settings.ini      # MOD配置文件
│       └── ersc_launcher.exe # MOD启动器
├── config.json               # 工具配置文件
└── EldenRingTool.exe         # 工具主程序
```

### 备份策略
1. **导入前备份**：当导入存档且目标目录已有内容时，自动创建备份
2. **直接覆盖选项**：用户可选择是否在导入时自动覆盖而不询问
3. **空目录保护**：存档目录为空时不创建不必要的备份
4. **备份区分**：自动备份和手动备份使用不同后缀，便于识别

## 关键判断逻辑

### 存档合法性检查
1. **基本检查**：存档文件夹是否存在
2. **结构适配**：
   - 情况A：存档文件夹包含`EldenRing`子文件夹
   - 情况B：存档文件夹直接包含存档文件
3. **关键验证**：
   - 查找17位数字的SteamID文件夹
   - 检查SteamID文件夹内是否存在`ER`开头的文件
   - `GraphicsConfig.xml`为可选文件，不影响合法性判断
4. **结果反馈**：
   - 合法：显示绿色状态和SteamID文件夹数量
   - 不合法：显示红色状态和具体问题描述

### MOD结构检查
1. **文件检查**：验证`SeamlessCoop`文件夹和`ersc_launcher.exe`文件是否存在
2. **配置读取**：从`ersc_settings.ini`直接读取联机密码和死亡惩罚设置
3. **状态反馈**：
   - 完整：绿色状态
   - 部分存在：橙色状态
   - 未找到：红色状态

## 已知问题与限制

1. **INI文件格式**：使用Python的`configparser`修改INI文件时，会丢失原文件的注释和空行格式。这是库本身的限制，目前没有简单的解决方案。

2. **文件权限**：Windows系统对正在使用的文件有访问保护，如果游戏正在运行，可能无法删除或覆盖存档文件。工具会尝试强制删除，但最安全的做法是关闭游戏后再操作。

3. **MOD兼容性**：目前仅针对无缝联机MOD进行了优化，其他MOD可能需要手动调整。

4. **路径权限**：某些系统目录（如Program Files）可能需要管理员权限才能写入。

5. **测试覆盖**：虽然工具经过了基本测试，但删除关键文件（如INI文件）后的行为可能未完全覆盖所有异常情况。

## 使用说明

### 首次使用准备
1. 将工具放在任意目录运行（建议不要放在系统保护目录）
2. 工具会自动创建必要的目录结构（`saves/`, `mods/`）
3. 将无缝联机MOD文件放入`mods/seamless_coop/`目录
4. 将存档文件放入`saves/`目录（支持多种结构）

### 日常使用流程
1. **启动工具**：运行`EldenRingTool.exe`
2. **检查路径**：确认所有路径都正确检测到（绿色状态）
3. **管理MOD**：
   - 点击"导入MOD文件"按钮导入MOD
   - 根据需要修改联机密码和死亡惩罚设置
4. **管理存档**：
   - 从下拉列表选择存档
   - 确认存档状态为绿色（合法）
   - 点击"导入存档"按钮
5. **启动游戏**：使用MOD启动器（`ersc_launcher.exe`）启动游戏

### 存档备份管理
- **自动备份**：每次成功导入存档前会自动创建备份（如果目标目录非空）
- **手动备份**：随时可以使用"导出存档"功能创建手动备份
- **备份管理**：所有备份按时间排序，最新的在最前面
